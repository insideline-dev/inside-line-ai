# Summary Tab ‚Äî Final UI Specification

## Data Source

The Summary tab pulls structured data from ALL evaluation agents. It does not have its own agent ‚Äî it aggregates fields from existing agent outputs. The only Summary-specific content is the Deal Snapshot paragraph, which is generated by the orchestrator (or a dedicated summary generation step) from the combined agent outputs.

---

## Section 1: Score Overview

### Overall Score Ring + Metadata Cards *(keep existing)*

**Left: Score Ring** ‚Äî large circular score (e.g., 23/100) with percentile below. Score is the weighted overall score calculated by the orchestrator function.

**Right: Metadata Cards** ‚Äî 2√ó4 grid of compact info cards showing deal metadata:

| Stage | Industry Group | Industry | Location |
|---|---|---|---|
| Round Size | Valuation (Post-money) | Raise Type | Lead Investor |

> Dev: These come from the submission/round data, not from eval agents. Keep existing implementation.

### Agent Score Radar Chart *(new)*

Below the score ring and metadata cards, a **radar/spider chart** plotting the score from every evaluation agent on its own axis. This is the single most information-dense visual on the page ‚Äî one glance shows where the deal is strong and where it's weak.

Axes (one per agent, rendered dynamically from available scores):
- Market
- Product
- Team
- Competitive Advantage
- Financials
- Traction
- Business Model
- GTM
- Legal
- Deal Terms
- Exit Potential

Each axis scales 0‚Äì100. The filled polygon shows the deal's shape. Color: single fill color with opacity (e.g., blue at 30% opacity with a solid blue border).

**Interaction:** Clicking any axis label navigates to that agent's tab.

**Missing scores:** If an agent hasn't completed or returned no score, that axis shows as 0 with a dashed line (not solid), and a small "pending" indicator on the label.

> Dev: ‚Üí Pull `scoring.overallScore` from each agent's output. All 11 agents use the same `scoring` wrapper structure. Render using a charting library (Recharts RadarChart, Chart.js radar, or D3). Axis labels are clickable ‚Äî navigate to the corresponding tab on click.

## Section 2: Deal Snapshot + Context Badges

### Deal Snapshot Paragraph *(keep existing, condensed)*

A generated paragraph summarizing what the company does, the raise details, and the key context. This is the "elevator pitch" view ‚Äî 3-4 sentences max.

> Dev: ‚Üí `dealSnapshot` (string). Generated by orchestrator from combined agent outputs. Keep existing implementation.

**Drop the bullet highlights** that currently appear below the paragraph ‚Äî they duplicate the Key Strengths section below.

### Context Badges Row *(new)*

A single horizontal strip of compact, clickable badge cards ‚Äî one per major evaluation dimension. Each badge shows a label, a key metric or status, and uses the agent's score to color-code the badge border (green >75, yellow 50‚Äì75, red <50). Clicking a badge navigates to that tab.

Badges (render left to right, skip any where data is unavailable):

1. **Market** ‚Äî TAM value + timing badge
   - Top line: TAM value (e.g., "$12B")
   - Bottom line: timing assessment as pill ("Right Time" green / "Early" yellow / "Late" red)
   > Dev: TAM ‚Üí `marketSizing.tam.value` from Market agent. Timing ‚Üí `marketGrowthAndTiming.timingAssessment` from Market agent. Map: "right_time" ‚Üí green, "slightly_early"/"slightly_late" ‚Üí yellow, "too_early"/"too_late" ‚Üí red.

2. **Product** ‚Äî tech stage
   - Single pill: tech stage (e.g., "Production" green / "MVP" yellow / "Concept" red)
   > Dev: ‚Üí `productOverview.techStage` from Product agent. Map: "production"/"scaling" ‚Üí green, "beta"/"mvp" ‚Üí yellow, "prototype"/"concept" ‚Üí red.

3. **Team** ‚Äî composition coverage + founder-market fit
   - Top line: "3/4 capabilities" (count of true booleans in team composition)
   - Bottom line: FMF score as small number (e.g., "FMF: 72")
   > Dev: Coverage ‚Üí count true values from `teamComposition.businessLeadership`, `.technicalCapability`, `.domainExpertise`, `.gtmCapability` from Team agent. FMF ‚Üí `founderMarketFit.score` from Team agent.

4. **Competitors** ‚Äî competitive gap + moat type
   - Top line: gap badge ("Leading" green / "Competitive" yellow / "Behind" red)
   - Bottom line: moat type pill (e.g., "Network Effects", "Technology")
   > Dev: Gap ‚Üí `competitivePosition.currentGap` from Competitive Advantage agent. Moat ‚Üí `moatAssessment.moatType` from Competitive Advantage agent.

5. **Financials** ‚Äî runway + projection credibility
   - Top line: runway badge (e.g., "24mo" green / "12mo" yellow) ‚Äî only if available
   - Bottom line: credibility pill ("Strong" / "Moderate" / "Weak")
   > Dev: Runway ‚Üí `keyMetrics.runway` from Financials agent (null if not available ‚Äî don't render badge). Credibility ‚Üí `projections.credibility` from Financials agent.

6. **Deal** ‚Äî valuation context
   - Top line: implied multiple (e.g., "20x ARR") ‚Äî only if available
   - Bottom line: premium/discount pill ("In Line" green / "Slight Premium" yellow / "Significant Premium" red)
   > Dev: Multiple ‚Üí `dealOverview.impliedMultiple` from Deal Terms agent (null if not available ‚Äî show score only). Premium ‚Üí `dealOverview.premiumDiscount` from Deal Terms agent. Map: "in_line"/"slight_discount"/"significant_discount" ‚Üí green, "slight_premium" ‚Üí yellow, "significant_premium" ‚Üí red, "insufficient_data" ‚Üí grey.

> Dev: Each badge card has a subtle left border colored by that agent's score (green/yellow/red). Clicking navigates to the tab. If an agent has no output yet, don't render that badge. The row should be horizontally scrollable on narrow viewports.

## Section 3: Return Profile *(new ‚Äî from Exit Potential agent)*

Three **scenario cards** displayed side by side, showing the exit analysis at a glance. This section answers the core investor question: "What's the return potential?"

### Card Layout (3 cards: Conservative / Moderate / Optimistic)

Each card shows:

- **Scenario label** as header ‚Äî "Conservative", "Moderate", "Optimistic"
- **Exit Valuation** ‚Äî large text (e.g., "$150M", "$500M‚Äì$800M")
- **MOIC** ‚Äî return multiple (e.g., "3.5√ó", "10.2√ó"). Color-coded: green >5√ó, yellow 3‚Äì5√ó, red <3√ó
- **IRR** ‚Äî annualized return (e.g., "25%", "45%"). Color-coded: green >30%, yellow 15‚Äì30%, red <15%
- **Timeline** ‚Äî years to exit (e.g., "5‚Äì7 years")
- **Exit Type** ‚Äî pill: "IPO" / "M&A" / "IPO or M&A"

Card background: subtle color tint per scenario ‚Äî grey for conservative, light blue for moderate, light green for optimistic.

> Dev: ‚Üí `exitScenarios[]` from Exit Potential agent (array of 3 objects, each with: scenario/string, exitType/string, exitValuation/string, timeline/string, moic/number, irr/number, researchBasis/string). Map scenario values ("conservative", "moderate", "optimistic") to card positions left-to-right. `researchBasis` is not shown on Summary ‚Äî it's available on the full Exit tab if we build one.

**If Exit Potential agent has no output:** show a muted placeholder: "Exit analysis pending" with the Exit Potential agent's score if available, or "Not yet evaluated" if not.

## Section 4: Key Strengths & Critical Risks

### Strengths & Risks Two-Column *(keep layout, improve sourcing)*

Same visual pattern as current ‚Äî green strengths left, red risks right. But the content should be **curated top items from across all agents**, not a dump from a single source.

**Left column: Key Strengths** ‚Äî top 4-5 strengths from across all agents. Each item tagged with its source agent as a small muted label (e.g., "Market", "Team", "Product").

**Right column: Critical Risks** ‚Äî top 4-5 risks from across all agents, tagged by source. Prioritize risks from agents with the lowest scores ‚Äî those are the most concerning.

> Dev: ‚Üí Aggregate `strengths` from all agents that have completed. Parse each agent's strengths string into individual items. Select the top 4-5 most impactful items across all agents (selection logic: prioritize items from highest-scoring agents for strengths, lowest-scoring agents for risks). Tag each item with the agent name. If a dedicated summary agent generates curated strengths/risks, use those instead.

### Critical Data Gaps *(new ‚Äî below the two columns)*

A compact list of **only the "critical" impact data gaps** from across all agents. Max 3-4 items. Each item shows:

- Gap description
- Source agent tag (muted label)
- Suggested action (muted text)

> Dev: ‚Üí Aggregate `dataGaps[]` from all agents. Filter to `impact: "critical"` only. Deduplicate similar gaps. Show max 4. Tag each with source agent name. If no critical gaps exist across all agents, don't render this subsection.

## Design Philosophy

The Summary tab is the **30-second decision view**. An investor lands here and within half a minute knows:

1. **Score radar** ‚Äî where is the deal strong and weak across all dimensions?
2. **Context badges** ‚Äî what does this company do, where does it sit, and what's the competitive/market position?
3. **Return profile** ‚Äî what's the realistic return potential across scenarios?
4. **Critical risks** ‚Äî what could kill this deal?

Everything else lives in the individual tabs. The Summary doesn't repeat tab content ‚Äî it pulls the single most important data point from each agent and presents them as a cohesive overview.

**Navigation principle:** Every visual element on the Summary tab is a portal to deeper analysis. Radar axes, context badges, and scenario cards all link to their respective tabs. The Summary is the map; the tabs are the territory.

Key visual elements:
- **Radar chart** ‚Äî the visual centerpiece, immediately communicates deal shape
- **Context badges row** ‚Äî compact horizontal strip of key metrics from each agent
- **Return scenario cards** ‚Äî three side-by-side cards with MOIC/IRR color-coded
- **Curated strengths/risks** ‚Äî tagged by source agent, not a single-agent dump
- **Critical data gaps** ‚Äî only dealbreakers, max 3-4 items
